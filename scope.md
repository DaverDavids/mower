# Project Document: Vision-Based Autonomous Lawn Mower

## 1. System Overview
An autonomous robotic lawn mower controlled by a Linux-based single-board computer (Atomic Pi). Instead of relying on a human operator or buried perimeter wires, the mower navigates outdoors using a hybrid tracking system that pairs a standard USB webcam (Visual SLAM) with a cheap IMU and wheel encoders to execute systematic, pre-planned cutting paths. The entire system is configured, monitored, and controlled via a responsive, browser-based Web UI over local Wi-Fi.

---

## 2. Hardware Architecture

### Core Computing & Sensors
*   **High-Level Brain:** Atomic Pi (Intel Atom x5-Z8350). Handles computer vision, sensor fusion, mapping, path planning, and hosts the Web UI server. Runs Ubuntu Linux + ROS 2.
*   **Low-Level Controller:** Microcontroller (Arduino Nano or RP2040). Handles real-time hardware tasks: reading wheel encoders and sending PWM signals to motor drivers.
*   **The "Eyes":** Standard USB Webcam. Mounted pointing slightly upward (10–15 degrees) to focus on permanent background landmarks (house, fence, large trees) rather than the changing grass.
*   **The Orientation Sensor:** MPU6050 (or similar cheap IMU). Measures rotational velocity and acceleration to maintain a heading.
*   **The Scale Anchor:** Dual Wheel Encoders (Optical or Magnetic). Measures exact physical wheel rotations to provide real-world distance metrics.

### Power & Actuation
*   **Power Supply:** Main Mower Battery (12V/24V) regulated down to a clean 5V/3A via a robust Buck Converter to power the Atomic Pi.
*   **Drive Actuators:** Dual DC Motor Drivers (H-Bridge) for differential steering (left/right wheel control).
*   **Blade Actuator:** Heavy-duty Relay or electronic speed controller (ESC) to toggle the cutting blade motor safely.

## 3. Software System Layout (ROS 2 Stack)

The system operates as a modular data pipeline built on **ROS 2 Humble / Iron**:

[USB Webcam] ──> (/image_raw) ──> [ ORB-SLAM3 Node ] ────┐│[Cheap IMU]  ──> (/imu_raw)   ──> [ IMU Filter Node ] ───┼─> [ robot_localization ] ──> [ Nav2 & Fields2Cover ]│        (EKF)                (Path & Move Commands)[Encoders]   ──> (Ticks)       ──> [ Microcontroller ] ──┘
1.  **Sensor Input Layer:**
    *   `usb_cam`: Publishes raw webcam images to `/image_raw`.
    *   `imu_driver` + `imu_filter_madgwick`: Filters raw IMU data to produce a stable heading angle.
2.  **Localization Layer (The Sensor Fuse):**
    *   `ORB-SLAM3`: Computes visual odometry by tracking pixel movement of distinct landmark features on your house/fences.
    *   `robot_localization` (Extended Kalman Filter): Fuses visual data, IMU data, and wheel encoder ticks. If wheels slip on wet grass, the camera corrects it; if shadows temporarily blind the camera, the encoders keep it moving straight.
3.  **Navigation & Planning Layer:**
    *   `Nav2 Costmap`: Maintains a digital grid of the environment.
    *   `Fields2Cover Plugin`: Takes your custom digital boundaries and automatically calculates optimized back-and-forth row mowing lines.
4.  **Hardware & UI Bridge:**
    *   `base_controller`: Converts movement paths (`/cmd_vel`) into raw text strings (e.g., `"L:100,R:100"`) sent via USB-Serial to the Arduino.
    *   `rosbridge_suite`: A WebSocket server running on the Atomic Pi that allows the Web UI to send commands and receive live map/video data using simple JavaScript.

## 4. Web UI System Design

To easily manage the mower from a phone or laptop over Wi-Fi, the Atomic Pi hosts a lightweight web server (Node.js/Express or Python Flask) serving a single-page dashboard. 

### Web UI Structure & Pages

#### Page 1: Dashboard & Mapping (Main View)
*   **Live Interactive Map:** Displays a top-down, canvas-rendered color map generated by the SLAM system.
    *   *Green Layer:* Grass/mowable safety zones.
    *   *Black/Red Dots:* Detected obstacles, house walls, and fences.
    *   *Mower Icon:* A live avatar showing the mower’s exact current position and heading in real time.
*   **Polygon Drawing Tool:** An interactive layer built over the map (using a library like Leaflet.js or HTML5 Canvas).
    *   *Inclusion Tool:* Allows you to tap/click to draw points, creating a closed polygon around your permitted grass layout.
    *   *Exclusion Tool:* Allows you to draw shapes around flowerbeds, trees, or birdbaths to treat them as permanent "No-Go" zones.
    *   *Save/Generate Button:* Serializes the polygon coordinates into a JSON format and pushes it to the ROS 2 path planner to generate the cut lanes.
*   **Mowing Status Display:** Shows current speed, area completed (sq ft), estimated time remaining, and current mowing lane tracking.

#### Page 2: Manual Control
*   **Virtual Joystick:** A touch-responsive on-screen thumbstick (e.g., nippleJS) for driving the mower manually over Wi-Fi. Perfect for the initial scanning run or driving it back to the garage.
*   **Hardware Toggles:**
    *   *Blade Motor Switch:* A prominent, safety-interlocked toggle switch to turn the cutting blade ON/OFF manually.
    *   *ESTOP (Emergency Stop):* A massive, bright red button that instantly forces a zero-velocity command to the motors and cuts power to the blade relay.

#### Page 3: Diagnostics & Debugging
*   **Camera Video Stream:** Displays a compressed, low-latency live view (`web_video_server`) from the USB webcam, showing exactly what landmarks the robot sees.
*   **Feature Point Overlay:** Toggles a visual overlay showing the active "keypoints" (tracking dots) ORB-SLAM3 is using to calculate positioning.
*   **Sensor Health Checks:** Simple green/red indicators showing connection status for the IMU, Arduino Serial link, Encoders, and SLAM engine.
*   **Live Sensor Telemetry Graph:** A real-time scrolling line graph showing raw IMU heading drift and wheel encoder tick consistency.
*   **ROS 2 Console Log Window:** A scrolling text terminal displaying the system's live error outputs, warnings, and system logs.

#### Page 4: Settings & Configuration
*   **Mower Dimensions:** Input fields for wheel diameter, wheel track width (distance between drive wheels), and cutting deck width to fine-tune steering math.
*   **Path Planning Overlap:** Sliders to adjust how close together the cutting rows are generated (e.g., a 10% to 30% pass overlap to avoid leaving uncut strips of grass).
*   **Speed Limits:** Max linear speed (meters/second) and max turning speed limits for autonomous operations.
*   **Wi-Fi Credentials:** Network manager configuration pane for switching between home Wi-Fi and an ad-hoc hotspot network generated by the mower itself when deep in the backyard.

## 5. The "Map & Draw" Workflow

You do not need to drive the robot along the exact perimeter to program it.

1.  **The Scanning Run:** Open the **Manual Control** page on your phone. Drive the mower via the virtual joystick down the center of the lawn *just once*. The webcam scans surrounding landmarks. ORB-SLAM3 projects this into a top-down, 2D visual map.
2.  **Drawing the Boundary:** Switch to the **Dashboard & Mapping** page on your laptop. Use the Polygon Drawing Tool to drop coordinate points directly onto the generated map:
    *   Draw your main inclusion polygon around the grass boundaries.
    *   Draw exclusion polygons around delicate landscape features.
3.  **Autonomous Execution:** Click "Generate Path". The `Fields2Cover` algorithm instantly populates your inclusion zones with row-by-row cutting tracks. Click "Start Mission" on the UI to let the robot execute the path.

## 6. Phased Development Strategy

*   **Phase 1 (The Desk Test):** Rig the Atomic Pi, webcam, and IMU on your desk. Install ROS 2, calibrate the webcam lens, and launch the Web UI server on local host. Verify that moving the sensors by hand updates the mower icon position on the Web UI map canvas.
*   **Phase 2 (The RC & Record Phase):** Assemble the physical mower chassis. Drive it manually around your yard using the Web UI virtual joystick with the blade disconnected. Ensure the Arduino reads wheel encoder ticks smoothly without losing serial connection.
*   **Phase 3 (Software Integration):** Refine your Kalman filter, generate your top-down map, and practice drawing boundaries and saving polygons directly inside the Web UI interface.
*   **Phase 4 (Autonomous Tuning):** Run the autonomous path planner at very low speeds with the blade disconnected. Ensure the mower respects the drawn Web UI boundaries and leaves a conservative 6-to-12-inch safety buffer zone near delicate flowerbeds to account for minor sensor drift.
