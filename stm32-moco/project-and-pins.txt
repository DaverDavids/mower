## Project Objective
The goal is to implement a 3-axis Brushless DC (BLDC) motor controller on a single STM32F103C8T6 Blue Pill development board. The system utilizes 18 digital outputs for phase and driver control, 9 digital inputs for Hall position feedback, and relies on USB CDC (Virtual COM Port) for communication with the Atomic Pi Linux host, plus USB DFU (BOOT0 jumper) for firmware updates. SWD debug (PA13/PA14) remains active for live debugging via ST-Link.

## Implementation Strategy

   1. Hardware Safety: Place 10kΩ pull-down resistors on the Motor 1 lines (A8-A10, B13-B15). This prevents short circuits when pins float during MCU boot or flashing. Motors 2 and 3 low-side enable pins (A15, B3, B5, B9, PC13, PC14) default LOW at boot — no enables fire until explicitly commanded.

   2. Frequency Generation: TIM1, TIM3, and TIM4 are configured with Prescaler=0 and ARR=1439, producing 25 kHz PWM at 72 MHz system clock (72,000,000 / 1440 = 50,000 half-periods → 25 kHz). This is above the audible range and provides 1440 steps of duty cycle resolution.

   3. Advanced Timing (Motor 1): Use TIM1's hardware dead-time engine (HAL_TIMEx_ConfigBreakDeadTime) to force a microsecond delay between high and low-side switching. Set active polarities to HIGH and Idle States to RESET to guarantee outputs drop to 0V when disabled. Motors 2 and 3 rely on external hardware shoot-through protection; their low-side enables are driven as GPIO outputs.

   4. Commutation Loop: Poll or interrupt-capture the 9 Hall input pins. Use the 3-bit binary state per motor to index into a 6-step commutation lookup table. The table sets PWM duty on the active high-side channel and asserts the corresponding low-side enable GPIO for each step.

   5. USB Communication: USB CDC Virtual COM Port (A11/A12) is used for bidirectional communication with the Atomic Pi Linux host. The STM32 receives ASCII or binary motor commands and returns Hall-derived position/speed feedback. The same USB port supports DFU firmware flashing when BOOT0 jumper is set to 1.

   6. Debug: SWD on PA13/PA14 remains active at all times. Use ST-Link for live variable inspection, breakpoints, and commutation step verification during bring-up.

## Motor Outputs (18 Pins Total)

### Motor 1 (Advanced Timer - TIM1)
Uses the line buffer. TIM1 provides automatic hardware dead-time generation.
- High-Side:  A8 (CH1), A9 (CH2), A10 (CH3)
- Low-Side:   B13 (CH1N), B14 (CH2N), B15 (CH3N)

### Motor 2 (General Purpose Timer - TIM3)
Hardware driver prevents shoot-through. Low-side driven by GPIO enables.
- High-Side PWM:     A6 (CH1), A7 (CH2), B0 (CH3)
- Low-Side Enables:  A15, B3, B5 (GPIO Output, default LOW)

### Motor 3 (General Purpose Timer - TIM4)
Hardware driver prevents shoot-through. Low-side driven by GPIO enables.
- High-Side PWM:     B6 (CH1), B7 (CH2), B8 (CH3)
- Low-Side Enables:  B9, PC13, PC14 (GPIO Output, default LOW)

------------------------------
## Hall Effect Inputs (9 Pins Total)
All configured as GPIO Input with internal pull-up resistors enabled.
- Motor 1 Halls: A0, A1, A2
- Motor 2 Halls: A3, A4, A5
- Motor 3 Halls: B10, B11, B12

## Clock Configuration
- HSE: 8 MHz external crystal (Blue Pill onboard)
- PLL: x9 → SYSCLK = 72 MHz
- USB prescaler: /1.5 → USB clock = 48 MHz
- APB1: 36 MHz | APB2: 72 MHz | TIM clocks: 72 MHz
