## Project Objective
Implement a 3-axis Brushless DC (BLDC) motor controller on a single
STM32F103C8T6 Blue Pill development board. The system uses 18 digital
outputs for phase and driver control, 9 digital inputs for Hall position
feedback, and USB CDC for runtime command/debug access.

## Timer Configuration

All three timers: Prescaler=0, ARR=1439, 72 MHz clock → 50 kHz PWM.

  TIM1  – Motor 1, advanced timer.  Channels CH1/CH2/CH3 + CH1N/CH2N/CH3N.
           Hardware dead-time via HAL_TIMEx_ConfigBreakDeadTime.
           Pull-down resistors (10kΩ) on all six Motor 1 pins (A8-A10, B13-B15)
           to prevent shoot-through on boot/flash.

  TIM4  – Motor 2, general-purpose.  Channels CH1(B6) / CH2(B7) / CH3(B8).
           Low-side controlled by GPIO enables: A15, B3, B5.
           Hardware driver prevents shoot-through; no dead-time needed in FW.

  TIM3  – Motor 3, general-purpose.  Channels CH1(B4) / CH3(B0) / CH4(B1).
           Low-side controlled by GPIO enables: B9, B11, B12.
           Hardware driver prevents shoot-through; no dead-time needed in FW.

------------------------------
## Implementation Notes

  1. All PWM channels initialised with Pulse=0.  Timers start but outputs
     stay LOW until Motor_Enable() is called.
  2. Motor_SafeAll() zeroes all PWM and GPIO enables immediately – safe
     to call from any context.
  3. Hall inputs use internal pull-ups.  Unconnected Halls will read HIGH
     (state 0b111 = invalid, motor stays stopped).
  4. 6-step commutation: poll Hall state → index COMMUT_FWD/REV table →
     set active high/low channel pair.
  5. Phase map (MAP command) remaps logical CH0/1/2 to physical channels
     to correct for motor wiring order without re-soldering.
  6. Firmware update: set BOOT0 jumper HIGH, power-cycle, flash with
     STM32CubeProgrammer over USB.

## CubeMX IOC – Required Changes

The current stm32-moco.ioc does NOT match this wiring.  Before regenerating
code, make the following changes in STM32CubeMX:

  Pins to REMOVE current assignment:
    PA6  – currently TIM3_CH1  → reassign to GPIO_Input (M3HA)
    PA7  – currently TIM3_CH2  → reassign to GPIO_Input (M3HB)
    PB0  – currently TIM3_CH3  → reassign to TIM3_CH3 (keep, correct)
    PB1  – currently GPIO_Output → reassign to TIM3_CH4 (M3HSC)
    PB4  – currently GPIO_Output → reassign to TIM3_CH1 (M3HSA, needs AFIO partial remap)
    PB6  – currently TIM4_CH1  → keep as TIM4_CH1 (M2HSA) - correct
    PB7  – currently TIM4_CH2  → keep as TIM4_CH2 (M2HSB) - correct
    PB8  – currently TIM4_CH3  → keep as TIM4_CH3 (M2HSC) - correct
    PB11 – currently GPIO_Input → reassign to GPIO_Output (M3LSB)
    PB12 – currently GPIO_Input → reassign to GPIO_Output (M3LSC)
    PC13 – currently GPIO_Output → not used (remove or leave)
    PC14 – currently GPIO_Output → not used (remove or leave)

  Summary of IOC changes needed:
    PA6   GPIO_Input  PULLUP   (M3HA)
    PA7   GPIO_Input  PULLUP   (M3HB)
    PB1   TIM3_CH4             (M3HSC – requires TIM3 partial remap in AFIO)
    PB4   TIM3_CH1             (M3HSA – requires TIM3 partial remap in AFIO)
    PB11  GPIO_Output          (M3LSB)
    PB12  GPIO_Output          (M3LSC)
    PC13  reset to default (no function)
    PC14  reset to default (no function)

  IMPORTANT – TIM3 Partial Remap:
    PB4=TIM3_CH1 and PB1=TIM3_CH4 require AFIO_MAPR TIM3 partial remap.
    In CubeMX: Pinout → PB4 → TIM3_CH1 will prompt you to enable the remap.
    This remap also moves TIM3_CH2 to PB5 – but PB5 is M2LSC (GPIO output).
    To avoid conflict, Motor 3 only uses CH1(B4), CH3(B0), CH4(B1);
    CH2 is intentionally unused.
